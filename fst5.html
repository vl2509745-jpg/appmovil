Depuracion y seguridad en el desarrollo de apps

Herramientas de depuración:

Son utilidades que permiten ejecutar la app paso a paso, revisar variables y detectar fallos de forma precisa para corregirlos rápidamente. Ejemplo: usar el debugger de Android Studio para ver dónde se detiene el código.

Uso de registros del sistema:

Los registros o logs muestran mensajes que la app genera mientras funciona, y sirven para identificar errores, comportamientos inesperados o información útil para el diagnóstico. Ejemplo: leer un log que diga “Error de conexión” para saber por qué no carga una pantalla.

Refactorización de código:

Es el proceso de reorganizar y mejorar el código sin cambiar su funcionalidad, haciéndolo más claro, eficiente y fácil de mantener. Ejemplo: dividir una función muy larga en funciones pequeñas y ordenadas.

Mecanismos de seguridad en Apps:

Incluyen permisos, archivos de configuración protegidos, cifrado de datos y autenticación con servicios externos, todo para evitar accesos no autorizados y proteger información. Ejemplo: pedir permiso para acceder a la ubicación y guardar tokens cifrados para un login con Google.
